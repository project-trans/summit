第三章 角色设计
为了进一步规范本组织项目群下的权限管理，现设计以下的几个角色：
1. 外部参与者（Outside collaborator）
• B - Basic, 提供批准（approve）合并请求的权限。邀请加入项目
的编辑群。申请要求为至少一个合并请求（pull request）获得合
并。
• A - Advanced, 相比 Basic 等级增加向主仓库推送开发分支的权
限，增加合并合并请求的权限。
2. 成员（Member）- 相比外部贡献者（A）增加主分支直接推送（Push）
权限，可以处理邮件。邀请加入本组织工作群。
3. 主管（Admin）- 负责项目的主力运营，相比成员增加邀请外部参与者
的权限。
4. 主席（Owner）- 拥有本组织的最高权限。
本组织应当重新梳理各编辑组人员。
4
第四章 旗下各 wiki 项目的改进
为优化旗下各 wiki 的服务，现决定对各项目进行改进。
1. 在 MtF.wiki 当前版本中为首页添加实质性的导览内容；
2. 重新梳理 issue 和 PR 的中的 label;
3. 单独设计全新的视觉解决方案；
4. 在 MtF.wiki 未来的新版本中使用子域名区分各地区站点，推进各地
区站点的自治管理；
5. 将 FtM.wiki 转移到本组织的 GitHub 组织下，但仍然缺乏活跃维护，
需要招募维护主力；
6. 简化和加快内容收录流程，促进快速更新。

第五章 域名
为简化支出与简化管理，现规定域名的管理。
1. 域名的管理权为 Owner 级别所有。
2. 继续持有和运营以下域名：
(a) MtF.wiki
(b) FtM.wiki
(c) RLE.wiki
(d) 2345.LGBT
(e) Project-Trans.org
3. 以下域名不再维护，直至中止服务前使用重定向通告提醒用户访问我
们的主域名：
(a) MtFwiki.cn
(b) MtFwiki.com
4. 其他域名不再维护。

第六章 社交媒体
本组织的核心社交媒体有：
1. Twitter: @MtFwiki
2. Telegram 频道: @MtFwiki
3. Mastodon: @MtFwiki@mastodon.social
以上几个 MtF.wiki 名义的媒体都是本组织层面的媒体，内容也并不完全是
MtF.wiki 内容和跨性别女性的内容，但是由于该品牌的知名度，仍然保留
使用该名称；
以上账号管理的原则是：
1. 管理权由 Admin 以上权限持有；
2. 多账号应该同步更新；
3. 内容发布应当在工作组内征求意见；
4. 原则上只发布与跨性别有关联的新闻。
应该充分使用 awesome trans 账号发布非严肃内容讨论。

1. 由于 Transland 由 MtF.wiki 前 Owner 可橙（Kenchiu）开设，故现在
本组织需要特别整理与其关系；

任何运营；

组织账号下，现决定将该域名转移给 Transland 现运营者翎澈 Rinfair.
